# Electronic Dice (STM32F411E-DISCO)

---

## 🇬🇧 English Version

### Project Description
This project is a digital implementation of a traditional 6-sided die, developed on the **STM32F411VET6** (Discovery Board). The device simulates a dice roll when the user button is pressed, featuring animated displays and synchronized sound effects.

### Features
* **Random Roll (1-6):** Activated by the blue `USER` button (PA0).
* **Roll Animation:** * Lasts approx. 1 second.
  * Rapidly changing digits on the 7-segment display.
  * Sequential flashing of onboard LEDs (LD3-LD6).
* **Audio Feedback (Buzzer):**
  * Initial "click" sound.
  * Rising pitch during the animation to build tension.
  * Distinct final tone once the result is settled.
* **True Randomness:** The random number generator seed is derived from **RTC** registers (seconds, minutes, hours) and system ticks, ensuring different results every time the device is powered on.

### Hardware Configuration
* **7-Segment Display (Common Anode):** Connected to **Port D** (PD0-PD3, PD8-PD11).
* **Buzzer:** Connected to **TIM1 (Channel 1)** using PWM for frequency control.
* **LEDs:** Onboard LEDs on PD12 (Green), PD13 (Orange), PD14 (Red), PD15 (Blue).

## 🇵🇱 Polska Wersja

### Opis Projektu
Projekt cyfrowego odpowiednika tradycyjnej kostki do gry, wykonany na mikrokontrolerze **STM32F411VET6** (Discovery Board). Urządzenie po naciśnięciu przycisku przeprowadza animowane losowanie liczby od 1 do 6, sygnalizując proces dźwiękiem oraz efektami wizualnymi.

### Funkcjonalności
* **Rzut kostką:** Aktywowany niebieskim przyciskiem `USER` (PA0).
* **Animacja losowania:** * Trwa ok. 1 sekundę.
  * Dynamiczna zmiana cyfr na wyświetlaczu 7-segmentowym.
  * Sekwencyjne miganie wbudowanych diod LED (LD3-LD6).
* **Efekty dźwiękowe (Buzzer):**
  * Startowy "klik".
  * Narastający ton podczas animacji, budujący napięcie.
  * Wyraźny sygnał końcowy po ustaleniu wyniku.
* **Prawdziwa losowość:** Ziarno (seed) generatora jest pobierane z rejestrów **RTC** oraz licznika ticków systemowych.

## Code Highlight: Buzzer PWM
The sound is generated by dynamically changing the `ARR` (Auto-Reload Register) to achieve specific frequencies:

```c
void buzzer_play_freq(uint16_t freq_hz, uint32_t duration_ms) {
    uint32_t timer_base_clock = HAL_RCC_GetPCLK2Freq();
    uint16_t new_period = (timer_base_clock / freq_hz) - 1;
    __HAL_TIM_SET_AUTORELOAD(&htim1, new_period);
    __HAL_TIM_SET_COMPARE(&htim1, TIM_CHANNEL_1, new_period / 2);
    HAL_TIM_PWM_Start(&htim1, TIM_CHANNEL_1);
    // ...
}
